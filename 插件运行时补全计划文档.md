# 插件运行时补全计划

## 背景

Flutter 重制版的插件模块已能安装/解析插件，但调用阶段不稳定。根因经过比对后明确三类：

- 运行时模型不等价：原版每个插件是常驻 JS 实例，重制版每次调用都重执行脚本。
- 包 shim 功能退化：`cheerio / crypto-js / big-integer / qs / he / axios / cookies` 等与真实包行为不对齐。
- 管理层行为偏离：`userVariables` 未接线、cookie 不进 HTTP 桥、版本降级无保护、错误语义和原版不一致。

本计划按 P0 → P3 分四档推进。每档都给出明确 TODO、预期产出和验证方法。原则参考 `doc/engineering_principles.md`。

## 约束与原则

- 不做静默失败：任何回退都要有诊断日志。
- 单文件 300 行上限，超过就按职责拆分。
- 平台差异只在 `core/runtime/` 和 `core/filesystem/` 内隔离，`features/` 层不感知平台。
- 每个 TODO 完成时在 `flutter_app/` 下跑一次 `flutter analyze` 和 `flutter test` 再推进。
- Windows 是主验证目标，Android 相关改动留到 P3。

## 参考项

- 原版入口：`example/MusicFreeDesktop/src/shared/plugin-manager/main/index.ts`
- 原版方法：`example/MusicFreeDesktop/src/shared/plugin-manager/main/plugin-methods.ts`
- 原版插件类：`example/MusicFreeDesktop/src/shared/plugin-manager/main/plugin.ts`
- 原版 polyfill：`example/MusicFreeDesktop/src/shared/plugin-manager/main/polyfill/`
- 原版类型：`example/MusicFreeDesktop/src/types/plugin.d.ts`
- 重制版运行时：`flutter_app/lib/core/runtime/`
- 重制版插件服务：`flutter_app/lib/features/plugins/`

## P0 · 止血（先让现存插件稳定）

目标：把"当前已装插件在重制版里表现不稳"这件事变成"表现可预期"。完成 P0 之后，依赖状态的插件（token、cookie、登录）能在一个会话内稳定工作。

### P0-1 · 运行时改为"每插件一实例 + 脚本只执行一次"

- [ ] 新增 `core/runtime/plugin_runtime_instance.dart`，表示一个已加载的插件 JS 实例。
  - 持有 `JavascriptRuntime`、插件脚本 hash、sourceUrl、实例化时刻的 manifest、per-instance 消息通道 id。
  - 暴露 `invoke(method, args, {userVariables, timeout})`、`dispose()`。
- [ ] 重构 `PluginRuntimeHost`：
  - 不再做 `inspectPlugin/invokeMethod` 两个一次性方法，而是 `loadInstance(script, ...)` 返回 `PluginRuntimeInstance`，由上层持有。
  - `PluginRuntimeAdapter` 保留，但内部代理到 instance。
- [ ] 每个 `PluginRuntimeInstance` 自己拥有一对 `inspect wrapper + invoke wrapper` 所需的 shim 与 `plugin` 闭包，避免跨实例互相污染。
- [ ] 消息通道名改为 `MusicFreeXxx#<instanceId>`，`onMessage` 按 instance 注册/注销；`sendMessage` 在 JS 侧拼接前缀。
- [ ] `PluginManagerService` / `PluginMethodService` 持有 `Map<storageKey, PluginRuntimeInstance>`：
  - `load()` 首次读到插件时创建实例。
  - 脚本 hash 变化或插件卸载时 `dispose()`。
- [ ] 删除下载侧的 "每次新建 Host + Isolate" 老路径，复用同一实例（或提供 `clone()` 用于 isolate，且 clone 也只在 isolate 内执行一次脚本）。
- [ ] 验证脚本：写一个 fake 插件，顶层 `let token = Math.random()`，`search` 返回 `token`。用 `flutter test` 断言连续三次调用返回相同值。
- [ ] 分析器修复：`flutter_js`、`pointycastle` 等红点须在本步消除（`flutter clean && flutter pub get`、必要时锁定子依赖）。

### P0-2 · 接通 userVariables

- [ ] 扩展 `PluginMetaRecord`：新增 `Map<String, String> userVariables`，`copyWith` 支持。
- [ ] 在 `plugin_meta.json` 里持久化；`PluginMetaRepository` 读写对齐。
- [ ] `PluginRuntimeExecutionContext` 的 `userVariables` 改为从 `PluginMetaRecord` 注入；`PluginRuntimeHost` 不再默认传空 map。
- [ ] `PluginManagerService.invokePluginMethod` 与 `PluginSearchExecutor.searchAllEnabled/searchPlugin` 在传参前读取当前插件的 `userVariables`。
- [ ] `PluginController` 新增 `updateUserVariables(plugin, Map<String, String>)`，走 meta 仓写入并触发 `reload`。
- [ ] Presentation：`plugin_detail_page.dart` 根据 `manifest.userVariables` 渲染表单（`key`、`name`、`hint`）并支持保存。
- [ ] 回归用例：对一个依赖 cookie/token 的插件（如需登录的源）做一次端到端——填写变量后 search 成功。

### P0-3 · `he.decode` 接真实实现

- [ ] 新增 `core/runtime/internal/plugin_runtime_html_entities_bridge.dart`，通过 `package:html` 的 entity map 或手写常用映射提供 `decode(text)` 和 `encode(text)`。
- [ ] 在 shim 里把 `he.decode / he.encode` 改为走 `MusicFreeHtmlEntities` 桥。
- [ ] Shim 侧补基础常量别名：`he.escape`（等价 `encode`）、`he.unescape`（等价 `decode`）。
- [ ] 单元测试：覆盖 `&amp; &lt; &#x4e2d; &#20013; &quot; &apos; &nbsp;` 七种常见实体。

### P0-4 · Cookie 与 HTTP 桥打通

- [ ] `PluginRuntimeStateBridge.handleCookies` 的 `set` 逻辑按 `(domain, path, name)` 唯一化，不再单纯按 `name` 覆盖。
- [ ] `PluginRuntimeHttpBridge._sendInIsolate`：
  - 请求发出前按 `uri.host` 匹配 `plugin_cookies.json`，拼装 `Cookie` header（尊重 path、secure、expires）。
  - 响应里解析 `Set-Cookie`，写回 cookie 存储。
- [ ] Cookie 存储从"per-plugin 单实例"拆成"per-plugin + per-domain"二级结构，避免多插件互串。
- [ ] Shim 侧：`@react-native-cookies/cookies` 的 `get/set/flush` 调到新桥接口；`flush` 实现持久化落盘。
- [ ] 单元测试：
  - cookies.set → axios.get 自动带 Cookie。
  - 响应 Set-Cookie → cookies.get 能读到。
- [ ] 对齐文档：`docs/HANDOFF_ZH.md` 或新增 `docs/PLUGIN_RUNTIME.md` 说明 cookie 生命周期。

### P0-5 · 方法语义回到原版

- [ ] `PluginMethodService.getTopListDetail` 改为异常时返回 `PluginTopListDetailResult(isEnd: true, topListItem, musicList: [])`，删除 `getTopListDetailSafe`，调用点直接用主方法。
- [ ] `getMediaSource` 回退策略改为：仅对"请求音质"重试 `retryCount` 次，不跨音质回退；错误时返回 `null`，UI 层由用户决定是否换音质。
- [ ] `_buildMediaSourceAttemptMessage / _buildMediaSourceFailureSummary / _buildQualityFallbackOrder` 按上一步的收敛结果清理。
- [ ] `download_media_source_resolver.dart` 的跨音质循环对齐到同一策略（或显式区分"下载需要音质降级"的产品规则，放到 download 模块内部，不入运行时）。
- [ ] 回归：用 fake 插件跑一遍 4 个场景——`getTopListDetail` 抛错、返回非法数据、正常数据、空列表，断言 UI 层不闪退、不无限 loading。

### P0-6 · 调用级超时与错误隔离

- [ ] `PluginRuntimeInstance.invoke` 增加 `timeout`（默认 15s，和原版 axios 默认值一致）。超时后抛 `PluginInvocationTimeoutException`，不污染 runtime 全局状态。
- [ ] 超时后自动把该插件标为 "warning"，诊断里记录最近一次超时，便于 UI 展示。
- [ ] 控制台日志 `MusicFreeConsole` 按 instance 聚合，运行期落盘到 `pluginLogsDirectory/<storageKey>.log`（ASCII 模式，最多 1MB，滚动一次）。

### P0 完成门槛

- `flutter analyze`、`flutter test` 全绿。
- 新增 fake 插件测试：token 状态稳定、`he.decode` 正确、cookie 能往返、超时可触发。
- 至少一个真实插件源（用 `D:\musicWEP\音乐源.json` 中一条）在搜索 / 排行榜详情 / getMediaSource 三个路径上本地手测通过。
- `HANDOFF_ZH.md` 更新到 P0 完成状态。

## P1 · 包 shim 向真实库靠拢

目标：不再因为 shim 残缺导致插件逻辑失败。完成 P1 后，绝大多数原版插件应该"不用改代码"就能跑。

### P1-1 · `cheerio` 重写为"一次解析 + JS 侧选择"

- [ ] Dart 侧只做一次 `html.parse(html)` 并序列化为紧凑 JSON 节点树（tag、attrs、children、text、innerHtml、outerHtml）。
- [ ] JS 侧实现 `Collection`，所有 `find / filter / each / map / eq / parent / parents / children / siblings / closest / next / nextAll / prev / prevAll / has / not` 在闭包里完成，不再来回桥。
- [ ] 支持 `.attr(name, value)`、`.addClass/removeClass/hasClass/toggleClass`、`.data()`、`.text(value)`、`.html(value)` 的写入路径（可只对 JS 镜像生效，无需回写 Dart 端）。
- [ ] `.each` 的回调里 `this` 绑定为当前元素（原版使用 `.call(node, i, node)`）。
- [ ] `load(html)` 返回的 `$` 同时支持 `$(selector)` 调用和 `$.html() / $.text()`。
- [ ] 基础 selector 支持：`tag / .class / #id / [attr] / [attr="v"] / [attr^="v"] / [attr$="v"] / [attr*="v"] / :first-child / :last-child / :nth-child(n) / descendant / > child / , union`。不支持的 selector 明确抛错，不静默返回空。
- [ ] 单元测试：覆盖链式选择、属性写入、`.each this`、嵌套 HTML 解析。

### P1-2 · `crypto-js` 算法补齐

- [ ] `AES.decrypt`（CBC / ECB / PKCS7，支持 `WordArray` 与 base64 字符串输入）。
- [ ] `mode.CFB / CTR / OFB`。
- [ ] `pad.NoPadding / ZeroPadding / Iso97971 / Iso10126 / AnsiX923`。
- [ ] `DES / TripleDES / RC4 / RC4Drop`。
- [ ] `SHA512 / SHA3 / HmacMD5 / HmacSHA512`。
- [ ] `enc.Latin1 / enc.Utf16 / enc.Hex`（把 `Hex` 从空对象补成真正的 `parse/stringify`）。
- [ ] `lib.WordArray.create / lib.WordArray.random`。
- [ ] `format.OpenSSL`（AES salt 模式字符串）。
- [ ] 所有算法都走 `pointycastle` 本地实现，`package:crypto` 只用于 MD5/SHA 系列。
- [ ] 单元测试：三对 `encrypt → decrypt` 回环 + Hmac 的 RFC 向量。

### P1-3 · `big-integer` 本地化 + API 完整

- [ ] 把当前"每次运算都过桥"的实现改为 JS 本地 BigInt 实现（基于 `BigInt` 原生类型 + 闭包）。只有 `modPow` 在 radix > 16 或超过阈值时回落到 Dart `BigInt`。
- [ ] API：`add / subtract / minus / multiply / divide / mod / modPow / modInv / gcd / pow / abs / negate / square / isZero / isNegative / isPositive / isEven / isOdd / isPrime / bitLength / sign`。
- [ ] 位运算：`and / or / xor / not / shiftLeft / shiftRight`。
- [ ] 比较：`eq / neq / lt / lte / gt / gte / compare / compareAbs`。
- [ ] 构造：`bigInt(x, radix)`、`bigInt.randBetween(min, max)`、`bigInt.one / zero / minusOne`。
- [ ] `toString(radix)`、`toJSNumber()`、`valueOf()`。
- [ ] 单元测试：RSA 加密流程的三步模幂、`shiftLeft + and` 的掩码、`gcd` 小向量。

### P1-4 · `qs` 嵌套与数组支持

- [ ] `stringify` 支持 `{a:{b:1}}` → `a%5Bb%5D=1`、数组 bracket (`arrayFormat: 'indices'|'brackets'|'repeat'|'comma'`)。
- [ ] `parse` 支持 `a[b]=1`、`a[]=1&a[]=2`、嵌套多层。
- [ ] 选项默认值对齐原 `qs`（ `allowDots: false`、`encode: true`、`arrayFormat: 'indices'`）。
- [ ] 单元测试：对照 Node 端 `qs` 产出做 10 组 in/out 比对（手动生成固定向量）。

### P1-5 · `axios` 行为补齐

- [ ] `interceptors.request / response` 的注册与链式执行。
- [ ] `CancelToken` + `AbortController` 双支持。
- [ ] `transformRequest / transformResponse`（数组形式）。
- [ ] `paramsSerializer`（函数或 `{ serialize }` 对象）。
- [ ] `auth: { username, password }` → 自动 `Authorization: Basic ...`。
- [ ] `maxRedirects`、`withCredentials`（后者配合 cookie 桥）。
- [ ] `validateStatus` 已有，补 `onDownloadProgress / onUploadProgress`（可先抛未实现，只保证不 crash）。
- [ ] 错误对象结构对齐：`error.response / error.request / error.config / error.isAxiosError / error.code`。
- [ ] `axios.all / axios.spread`（简单别名 Promise.all）。
- [ ] 单元测试：拦截器链、取消请求、basic auth、带 cookie 的请求。

### P1-6 · `dayjs` 补扩展

- [ ] `diff(other, unit)`、`startOf / endOf`、`isBefore / isAfter / isSame`。
- [ ] `locale(tag)` 允许切换，但只实现 `en / zh-cn` 两种月份/星期文本。
- [ ] format token 扩展：`A / a`（上下午）、`SSS`（毫秒）、`Z / ZZ`（时区偏移，按 UTC+08:00 固定本地）、`D / Do`（日）、`YY`。
- [ ] `unix(seconds)` 构造（已有）、`utc()` / `local()` 占位（内部固定本地，返回 this）。
- [ ] 单元测试：十组 format 向量 + diff/startOf 小测。

### P1-7 · Cookies / WebDAV API 扩展

- [ ] cookies 补 `getAll / clearAll / clearByName / clearByDomain`。
- [ ] webdav 补 `getFileContents / putFileContents / createDirectory / exists / stat / moveFile / copyFile / deleteFile`。
- [ ] WebDAV 的 `getFileContents` 支持 `{ format: 'text' | 'binary' }`，二进制返回 `Uint8Array` 等价结构。
- [ ] 单元测试：用本地 http server 模拟 PROPFIND/PUT/MOVE 各走一遍。

### P1 完成门槛

- `flutter test` 下覆盖 P1 新增全部单元测试。
- 把 `D:\musicWEP\音乐源.json` 列的每一条插件至少跑通 `search / getTopLists / getTopListDetail / getMediaSource`（允许部分源因站点侧失效而失败，但必须报出明确错误，不是静默空）。
- shim 规模控制：单文件不超过 300 行，`plugin_runtime_package_shims.dart` 拆分为 `shims/{cheerio, crypto, bigint, axios, qs, dayjs, he, cookies, webdav}.dart`。

## P2 · 管理层与工程完善

目标：把运行时外围的"版本、订阅、诊断、清理"等问题闭环，具备长期可维护性。

### P2-1 · 版本降级保护

- [ ] 引入 `package:pub_semver` 已用，补一个 `compareVersions(String a, String b)` 工具（解析失败时按字符串比较，避免抛错）。
- [ ] `PluginInstallationCoordinator.installScript`：若同 storageKey 已存在且新版本 < 旧版本，抛 `PluginDowngradeBlockedException`，信息文案对齐原版"已安装更新版本的插件"。
- [ ] UI 层弹出可覆盖选项（"强制覆盖"），仅在用户主动确认时允许继续（避开静默覆盖）。
- [ ] 单元测试：降级被拦、同版本静默成功、升级正常。

### P2-2 · manifest 字段补齐

- [ ] `PluginManifest` 扩字段：`primaryKey: List<String>`、`cacheControl: String?`、`hints: Map<String, List<String>>`、`defaultSearchType: String?`。
- [ ] `plugin_runtime_inspect_wrapper_builder.dart` 把这些字段从 `plugin` 原样 dump。
- [ ] `PluginMethodService`、搜索 / 下载流程按 `primaryKey` 去重（对齐原版 `getPluginPrimaryKey`）。
- [ ] `cacheControl` 和 `getMediaSource` 的 URL 缓存策略挂钩（为 P3 的 Android 做准备，当前 Windows 侧可先读不写）。
- [ ] UI `plugin_detail_page.dart` 展示 hints。

### P2-3 · 订阅与安装流程收敛

- [ ] 订阅 URL 统一 `addRandomHash(url)`（`?_=${timestamp}`）防 CDN 缓存，和原版对齐。
- [ ] 订阅 JSON schema 支持 `{ name, url }`，保留子插件的人类可读名。
- [ ] `PluginInstallationCoordinator.installScript` 重命名落盘文件：`<storageKey>_<version>_<hashShort>.js`，方便肉眼排障。
- [ ] `loadAllPlugins` 发现重复 hash 时保留最早那个，不删文件；重复记入诊断日志。
- [ ] 订阅刷新过程里每个子插件失败独立记录，不阻塞其他子插件安装（已有，但要补细节日志）。

### P2-4 · 诊断 / 日志体系

- [ ] 新增 `core/runtime/plugin_runtime_logger.dart`：按 `storageKey` 切分，单文件 1MB 滚动，保留最近两份。
- [ ] 每次 `invoke` 前后记录：method、duration、logs、error。
- [ ] `PluginDiagnostics` 新增 `lastInvokeAt`、`lastInvokeErrorMessage`、`invokeFailureCount`。
- [ ] UI `diagnostics_page.dart` 展示最近 50 条调用记录（按插件聚合）。
- [ ] 提供导出诊断包功能：把 `plugin_meta.json` + 日志打包成 zip，便于用户反馈。

### P2-5 · 运行时稳定性优化

- [ ] `PluginRuntimeInstance` 内部对 JS 消息通道做并发互斥（同一 instance 的 `invoke` 串行化），避免 QuickJS 异步作业顺序错乱。
- [ ] 给长时间空闲的 instance 做"软卸载"：超过 30 分钟未调用则释放 QuickJS，下次调用时懒加载回来。
- [ ] 为 HTTP 桥增加 `maxConcurrentRequests = 16` 全局限流，避免 QuickJS 大量并发 Promise 打爆。
- [ ] 为 `sendMessage` 返回值里可能包含的大对象（比如 cheerio 节点树）增加体积阈值告警（> 5MB 写日志）。

### P2-6 · 文档与规范

- [ ] 新建 `flutter_app/docs/PLUGIN_RUNTIME.md`：
  - 模块分层图
  - 调用时序（install → load → invoke → dispose）

- [ ] 更新 `HANDOFF_ZH.md` 和 `HANDOFF_2026-04-06.md` 中过时的描述。

### P2 完成门槛

## P3 · Android 适配准备

目标：在 P0-P2 的基础上把插件模块移植到 Android，并把 desktop-only 的组件换成 Android 侧方案。

### P3-1 · 运行时引擎对齐

- [ ] `PluginRuntimeHost._createRuntime` 在 Android 下走 `getJavascriptRuntime()`（JavaScriptCore），确认和 QuickJS 的行为一致；差异点用 `PluginRuntimeCapabilities` 抽象出来。
- [ ] JavaScriptCore 与 QuickJS 在 `BigInt`、`Proxy`、`Symbol.iterator`、`URL` 原生支持上的差异要跑 JS 侧的 feature detection，缺失的走 polyfill。
- [ ] Android 上验证 instance 软卸载与并发互斥是否仍稳定（JavaScriptCore 的 GC 行为不同）。

### P3-2 · 文件 / 路径 / 权限

- [ ] `AppPaths` 已用 `getApplicationSupportDirectory`，在 Android 上对应 `/data/data/<pkg>/files`，需要验证。
- [ ] 外部存储下的本地音乐导入（`importMusicItem / importMusicSheet`）接入 `permission_handler`（`MANAGE_EXTERNAL_STORAGE` / `READ_MEDIA_AUDIO`）。
- [ ] 本地音乐扫描改为 MediaStore API 走 platform channel，避免 path 访问被系统限制。

### P3-3 · HTTP / Cookie 调整

- [ ] Android 下 `badCertificateCallback` 行为与 Dart VM 一致，但需要验证 TLS 1.3 与 ALPN。
- [ ] Cookie 持久化位置确认可写；跨 app 启动后还能带 cookie。

### P3-4 · 播放器 / 歌词窗

- [ ] 桌面歌词窗替换为 Android `SYSTEM_ALERT_WINDOW` overlay 或通知栏媒体控制。
- [ ] `desktop_multi_window` 用法做条件编译，Android 路径走空实现 stub。
- [ ] `window_manager` / `tray_manager` 同样走条件编译。

### P3-5 · 构建 & CI

- [ ] 在 `flutter_app/android/` 完成 Gradle/Kotlin 最小配置，跑 `flutter build apk --debug`。
- [ ] `flutter test` 保证不依赖 desktop-only 包。
- [ ] CI 增加 Android build 作为 matrix 的一个目标（如已配置 Windows，只是再加一条）。

### P3 完成门槛

- `flutter build apk --debug` 可产出 apk。
- 至少三个真实插件在 Android 真机上跑通 `search / getMediaSource / 播放`。
- 桌面歌词和托盘的 desktop-only 组件在 Android 下不报错，功能用替代方案或留白（不是崩溃）。

## 迭代验证方式

每档完成后统一走一遍：

1. `flutter analyze`
2. `flutter test`
3. `flutter run -d windows` 手测一条真实插件源
4. 更新 `HANDOFF_ZH.md` 的"当前完成情况"段

## 风险登记

- 运行时重构涉及跨模块接口，单次改动较大。拆小步提交，每步保证编译过。
- 包 shim 补齐量大，容易越写越复杂。坚持"单文件 300 行 + 按依赖拆分"。
- Android 适配可能暴露 `flutter_js` / `media_kit` / `desktop_multi_window` 的平台不兼容，需要预留 buffer。

## 建议启动顺序

1. P0-1（运行时单例重构）完成后再并行 P0-2 到 P0-6。
2. P1 的顺序可并行，但 `cheerio` 和 `crypto-js` 最影响插件通过率，先排。
3. P2 依赖 P1 完成度，不急。
4. P3 在 P2 完成门槛达成后启动。
