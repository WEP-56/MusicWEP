## 4. 当前已完成但仍需持续打磨的点

### 4.1 深色模式

深色模式已经可用，但仍不是全量页面完成态。

已明显修过的区域：

- 主框架
- 侧边栏
- 顶栏
- 底部播放器
- 右侧播放列表

- 热门歌单

- 插件管理
- 最近播放
- 下载管理
- 歌曲详情页

仍可能残留的问题：

- 某些业务页仍有固定浅色块
- 某些旧页面还存在固定深色文字，没有完全切到 `Theme.of(context).colorScheme.*`
- 一些中文文案仍有历史错码

建议继续处理方式：

- 优先改公共组件和页头/表头/卡片，不要先零散修某一行文字。
- 固定色优先替换为：
  - 主文本：`theme.colorScheme.onSurface`
  - 次文本：`theme.colorScheme.onSurfaceVariant`
  - 面板底：`theme.colorScheme.surface` / `surfaceContainerLow` / `surfaceContainerHigh`
  - 强调色：`AppTheme.colorsOf(context).accent`

## 7. 已知问题 / 风险

### 7.1 偶发 Stack Overflow

用户最近偶发过一次：

- `Unhandled Exception: Stack Overflow`
- 栈只有 `Future._propagateToListeners...` 重复

当前状态：

- 未稳定复现
- 这次会话中没有拿到明确触发路径

高可疑区域：

- 主题切换后的异步通知链
- 窗口桥接 / 桌面歌词窗通信
- 某些 `showDialog` / `Navigator.pop` 后再次触发异步状态更新

建议下一会话如果用户再次遇到，优先追问：

- 触发前做了什么操作
- 是否与主题切换有关
- 是否与歌词窗开关有关
- 是否与右侧播放列表或弹窗有关

### 7.2 文案错码仍残留

虽然这几轮顺手修了一部分，但仓库里仍有不少历史错码中文。后续不要全局盲修，应该：
